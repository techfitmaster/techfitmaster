> CTO 设计文档 v1.0 | 2026-03-04

---

## 背景与定位

**目标市场：** 俄罗斯
**核心痛点：** 俄罗斯用户访问 OpenAI / Claude 受限（支付封锁 + 访问限制）
**商业机会：** 提供稳定的 AI API 中转服务，解决俄罗斯用户的访问和支付问题
**自动获客：** 针对俄罗斯游戏/开发者社群，自动挖掘有 AI 需求的用户

## 一、AI 中转站（818-ru-ai-svc-proxy）

### 核心价值主张
```
俄罗斯用户 → 无法直接用 OpenAI/Claude
           → 818 中转站提供：

             3. 稳定低延迟（就近节点）
```

### 俄罗斯市场特殊设计

**支付方式（关键差异化）：**
```
优先级：
1. USDT / 加密货币    → 最主流，无地域限制
2. Telegram Stars     → Telegram内置支付，俄罗斯用户常用
3. 卢布转账（P2P）   → 人工审核，风险较高，后期考虑
```

**接入方式：**
```
方式A：Telegram Bot（推荐首发）
  → 俄罗斯用户极度依赖Telegram
  → Bot可完成：注册/充值/查余额/API调用
  → 零App开发成本，快速上线

方式B：Web控制台（Phase 2）
  → 俄语界面
  → 需要CDN节点在俄罗斯可访问地区
```

**定价策略：**
```
充值制（卢布/USDT）
$5   → 500 积分
$20  → 2200 积分（10% bonus）
$50  → 6000 积分（20% bonus）

模型定价（参考竞品，略低10-15%）：
GPT-4o     $0.015/1K Token
Claude     $0.018/1K Token
DeepSeek   $0.003/1K Token（引流款）
```

**竞争优势：**
```
vs 现有俄罗斯中转站：
✅ Telegram Bot 即开即用（他们多数只有网页）
✅ DeepSeek 超低价引流
✅ 支持 USDT（部分竞品不支持）
✅ 中国节点低延迟（部分模型访问更快）
```

### Telegram Bot 核心功能
```
/start     → 注册 + 生成API Key
/balance   → 查询余额
/recharge  → 充值（USDT地址 / Telegram Stars）
/models    → 查看可用模型和价格
/usage     → 本月用量统计
/help      → 帮助（俄语）

自动触发：
余额<20%  → 自动发低余额提醒
API调用失败 → 自动说明原因
```

## 三、冷启动策略

### 第一步：Telegram Bot先行（Week 1-2）
```
1. 搭建基础Bot（注册/充值/API调用）
2. 手动邀请5-10个种子用户测试
3. 收集反馈，修复问题
```

### 第二步：小批量获客（Week 3-4）
```
1. 加入10个俄罗斯AI相关Telegram群
2. 观察并参与讨论（人工，建立信任）
3. 准备好话术模板
4. 验证转化率
```

### 第三步：自动化扩张（Week 5-6）
```
1. 部署获客Agent，接管关键词监控
2. 自动触达 + 人工审核话术
3. 目标：每周新增50+注册用户
```
