### 二、技术栈选型

| 层级 | 技术 | 理由 |
|------|------|------|
| **现有后端** | Java + Spring Boot | 818勇士助手存量，稳定可靠 |
| **业务后端** | Java + Spring Boot | 复用现有技术，减少学习成本 |
| **AI 服务** | Python + FastAPI | LLM 生态丰富、易集成 |
| **AI 中转** | Go + Redis | 高并发、限流、缓存 |
| **API 网关** | Kong / Nginx | 统一入口、限流、鉴权 |
| **数据库** | MySQL 8.0 + Redis | 稳定 + 高速缓存 |
| **消息队列** | Redis Stream | 轻量、足够用 |
| **部署** | Docker + 阿里云 | 弹性伸缩 |
| **CI/CD** | GitHub Actions | 免费、私有仓库无限 |

### 四、关键设计决策

| 决策点 | 选型 | 原因 |
|--------|------|------|
| **统一鉴权** | JWT + Redis | 多端统一、session 共享、支持分布式 |
| **支付路由** | 配置化 + 策略模式 | 微信商户迁移无感、热插拔 |
| **AI 接入** | 抽象层 + 适配器模式 | 轻松切换 LLM 提供商（OpenAI/Claude/国产） |
| **账号交易** | 保守设计 | 资金安全 > 性能，事务优先 |
| **服务通信** | HTTP REST | 简单可靠，后续可升级 gRPC |
| **缓存策略** | 多级缓存 | 本地缓存 + Redis，热点数据加速 |

## 十二、竞品分析

### 1. 游戏辅助工具 (818勇士助手)
| 竞品 | 特点 | 优势 | 劣势 |
|------|------|------|------|
| 蜂窝助手 | 品类多、用户多 | 品牌认知 | 收费贵、客服响应慢 |
| 叉叉助手 | 脚本丰富 | 社区活跃 | 稳定性一般 |
| 飞火手游 | 定制化强 | 技术强 | 品类少 |

> **策略：** 聚焦特定游戏、做深体验、定价灵活

### 2. 账号交易
| 竞品 | 特点 | 优势 | 劣势 |
|------|------|------|------|
| 交易猫 | 平台大、品类全 | 流量大 | 抽成高、服务费贵 |
| 7881 | 交易快 | 手续费低 | 客服一般 |
| 私下交易 | 灵活 | 无平台费 | 风险高 |

> **策略：** 专注游戏账号 + 担保交易 + 快速变现

### 4. AI赋能 (外贸电商)
| 竞品 | 特点 | 优势 | 劣势 |
|------|------|------|------|
| Jasper AI | 营销文案强 | 成熟 | 贵、无本地化 |
| Copy.ai | 入门简单 | 易用 | 深度不够 |
| 国内AI工具 | 本地化 | 中文好 | 定制弱 |

> **策略：** 垂直外贸场景 + 私有化部署 + 订阅制
