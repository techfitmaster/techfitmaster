# SPEC - [T1] 申诉回复语自动化闭环

## [T1] 申诉回复语自动化配置

**目标**：为申诉处理流程提供可配置的回复模板，减少客服人工输入成本

**业务规则**：
- 管理员可在后台配置常用申诉回复模板（CRUD 操作）
- 每个模板包含：申诉类型、回复内容、是否启用
- **小程序**申诉审核页面可选择并快速引用模板
- **管理后台**申诉处理页面也可选择并快速引用模板
- 支持多种申诉类型（账号冻结、订单问题、结算争议等）

**涉及端**：
- [x] 后端 API（CRUD 接口 + 数据库表）
- [x] 管理后台（模板配置页面 + 申诉处理页集成模板选择）
- [x] 小程序（申诉审核工具集成）

**BDD 场景**：

### 场景组 1：模板管理（管理后台）
| Given（前置条件） | When（操作） | Then（预期结果） |
|------------------|-------------|-----------------|
| 管理员登录后台 | 访问申诉回复配置页面 | 显示所有已配置的模板列表 |
| 管理员创建新模板 | 选择申诉类型并输入回复内容 | 模板保存成功，状态为"启用" |
| 管理员编辑模板 | 修改回复内容并保存 | 模板更新成功 |
| 管理员删除模板 | 点击删除按钮并确认 | 模板从列表中移除 |
| 管理员禁用模板 | 将模板状态改为"禁用" | 小程序和后台处理申诉时不可见 |

### 场景组 2：使用模板处理申诉（小程序）
| Given（前置条件） | When（操作） | Then（预期结果） |
|------------------|-------------|-----------------|
| 客服在小程序处理申诉 | 在申诉审核页选择回复模板 | 回复框自动填充模板内容 |
| 客服选择已禁用的模板 | 尝试引用模板 | 该模板不在可选列表中 |
| 客服填充模板后 | 可手动修改回复内容 | 支持自定义编辑 |

### 场景组 3：使用模板处理申诉（管理后台）
| Given（前置条件） | When（操作） | Then（预期结果） |
|------------------|-------------|-----------------|
| 管理员在后台处理申诉 | 打开申诉详情页并选择回复模板 | 回复框自动填充模板内容 |
| 管理员选择模板并发送 | 点击"提交回复"按钮 | 申诉状态更新为"已处理"，回复发送给用户 |
| 管理员在后台查看申诉列表 | 筛选"待处理"状态的申诉 | 显示所有未处理的申诉 |

**技术约束**：
- 回复内容字段长度限制：500 字符
- 申诉类型需与现有 `appeal_type` 枚举对齐

**数据库变更**：
```sql
-- V2.0.65.sql
CREATE TABLE `appeal_reply_config` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `appeal_type` varchar(50) NOT NULL COMMENT '申诉类型',
  `reply_content` varchar(500) NOT NULL COMMENT '回复内容',
  `enabled` tinyint(1) DEFAULT 1 COMMENT '是否启用',
  `created_at` timestamp DEFAULT CURRENT_TIMESTAMP,
  `updated_at` timestamp DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_appeal_type` (`appeal_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='申诉回复模板配置';
```
