# DAS 数据库自治服务配置指南

## 1. 开通 DAS

### 控制台操作
1. 访问 DAS 控制台：https://hdm.console.aliyun.com/
2. 点击「立即开通」
3. 选择「基础版」（¥60/实例/月）
4. 确认订单

## 2. 接入 RDS 实例

### 2.1 添加实例
1. DAS 控制台 → 「实例管理」→ 「添加实例」
2. 选择你的 RDS MySQL 8.0 实例
3. 点击「接入」

### 2.2 开启功能
接入成功后，建议开启以下功能：

- ✅ **性能洞察**：实时 SQL 分析
- ✅ **慢 SQL 分析**：自动识别慢查询（执行时间 >1s）
- ✅ **SQL 优化建议**：智能索引推荐
- ✅ **锁分析**：死锁、锁等待检测
- ✅ **容量预测**：磁盘/连接数趋势预测

## 3. 配置慢 SQL 告警

### 3.1 创建告警规则
1. DAS 控制台 → 「告警管理」→ 「创建告警规则」
2. 选择实例 → 选择指标

**推荐告警规则：**

| 告警名称 | 指标 | 阈值 | 频率 | 通知方式 |
|---------|------|------|------|---------|
| 慢 SQL 突增 | 慢 SQL 数量 | >10 条/5分钟 | 5分钟 | 钉钉 + 短信 |
| SQL 执行超时 | 平均执行时间 | >2s | 5分钟 | 钉钉 |
| 锁等待 | 锁等待时间 | >5s | 1分钟 | 钉钉 + 短信 |
| 连接数告警 | 连接使用率 | >85% | 5分钟 | 钉钉 |

### 3.2 告警通知配置
1. DAS 控制台 → 「告警管理」→ 「联系人管理」
2. 添加钉钉机器人 Webhook
3. 添加手机号（接收短信）

**钉钉机器人 Webhook 获取方式：**
1. 钉钉群 → 群设置 → 智能群助手
2. 添加机器人 → 自定义机器人
3. 复制 Webhook URL

## 4. SQL 优化工作流

### 4.1 查看慢 SQL
DAS 控制台 → 「性能洞察」→ 「慢 SQL 分析」

### 4.2 分析 SQL
点击慢 SQL → 查看：
- 执行计划
- 索引使用情况
- 表扫描行数
- 优化建议

### 4.3 优化示例

**原始 SQL（慢查询）：**
```sql
SELECT * FROM room 
WHERE game_server = 'CN' AND status = 'WAITING'
ORDER BY create_time DESC 
LIMIT 10;
```

**DAS 建议：**
- ❌ 缺失索引：`idx_game_server_status_time`
- ⚠️ 全表扫描：扫描 10万行

**优化后：**
```sql
-- 1. 添加复合索引
CREATE INDEX idx_game_server_status_time 
ON room(game_server, status, create_time DESC);

-- 2. SQL 保持不变（会自动使用索引）
SELECT * FROM room 
WHERE game_server = 'CN' AND status = 'WAITING'
ORDER BY create_time DESC 
LIMIT 10;
```

**优化效果：**
- 执行时间：2.5s → 15ms
- 扫描行数：100,000 → 10

## 7. 常见问题

### Q: DAS 会影响数据库性能吗？
A: 不会。DAS 通过只读账号采集监控数据，对生产环境几乎无影响。

### Q: 慢 SQL 阈值可以调整吗？
A: 可以。DAS 控制台 → 「设置」→ 「慢 SQL 阈值」，默认 1s，可调整为 0.5s 或 2s。

### Q: 能监控主从延迟吗？
A: 可以。DAS 会自动监控主从复制延迟，并在延迟 >5s 时告警。

### Q: 支持 SQL 审计吗？
A: 基础版不支持，需要升级到专业版。但可以通过 RDS 自带的「SQL 洞察」功能实现（按量付费）。
