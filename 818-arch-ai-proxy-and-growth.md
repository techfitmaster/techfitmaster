## 一、AI 中转站（818-ai-svc-proxy）

### 定位
游戏玩家/工作室专属 AI API 接入服务，全程零人工干预自动盈利。

### 系统架构

```
┌─────────────────────────────────────────────────────┐
│                    用户层                            │
│         Web控制台 │ API Key │ SDK                   │
└────────────────────┬────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────┐
│              网关层 (Go + Gin)                       │
│  鉴权(API Key) │ 限流(Redis) │ 路由 │ 日志          │
└──────┬─────────────┬──────────────┬─────────────────┘
       │             │              │
┌──────▼──────┐ ┌────▼────┐ ┌──────▼──────┐
│  模型路由   │ │  计费   │ │  健康检测   │
│  智能选择   │ │  引擎   │ │  自动熔断   │
│  负载均衡   │ │  Token  │ │  降级切换   │
└──────┬──────┘ └────┬────┘ └─────────────┘
       │             │
┌──────▼─────────────▼────────────────────────────────┐
│              模型供应商层                            │
│  OpenAI │ Claude │ DeepSeek │ Gemini │ 国产模型     │
└─────────────────────────────────────────────────────┘
       │
┌──────▼──────────────────────────────────────────────┐
│              存储层                                  │
│  MySQL(用户/账单) │ Redis(限流/缓存) │ ClickHouse   │
│                               (调用日志分析)         │
└─────────────────────────────────────────────────────┘
```

### 核心业务流程
```
用户注册 → 自动开通API Key → 充值(微信/支付宝) → 自动扣费
→ 额度预警(80%发通知) → 耗尽自动停止
全程零人工
```

### 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 网关层 | Go + Gin | 高并发，低延迟 |
| 计费引擎 | Go | 精确到Token级别 |
| 控制台 | Vue3 | 用户自助管理 |
| 存储 | MySQL + Redis | 用户数据 + 实时限流 |
| 日志 | ClickHouse | 亿级调用分析 |

### 差异化：游戏垂直场景
```
预置游戏场景 Prompt 模板
│
├── 游戏攻略生成 API
├── 账号估值 API
├── 客服话术 API
└── 脚本生成 API（按场景定价，比通用贵3-5倍）
```

### 定价模型
```
充值制（类电话卡）
¥10  → 1000 积分
¥50  → 5500 积分（10% bonus）
¥200 → 25000 积分（25% bonus）

调用成本：
GPT-4o   10积分/千Token
Claude   12积分/千Token
DeepSeek 2积分/千Token（引流款）
```

### 数据库设计
```sql
t_ai_user          -- 用户表（API Key、余额、套餐）
t_ai_recharge      -- 充值记录
t_ai_usage         -- 用量记录（按Token计）
t_ai_request_log   -- 请求日志（ClickHouse）
t_ai_model_config  -- 模型配置（价格、开关、权重）
t_ai_prompt_tpl    -- 游戏场景Prompt模板
```

## 二、AI 自动获客系统（818-growth-agent-acquire）

### 定位
游戏社群自动扫描 → 线索识别 → 触达 → 转化，替代人工销售。

### 系统架构

```
┌─────────────────────────────────────────────────────┐
│              数据采集层（Agent）                     │
│                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │ 论坛爬虫 │  │ 关键词监 │  │  竞品用户扫描    │  │
│  │ NGA/贴吧 │  │ 控(定时) │  │  (蜂窝/叉叉用户) │  │
│  └────┬─────┘  └────┬─────┘  └────────┬─────────┘  │
└───────┼─────────────┼────────────────┼─────────────┘
        │             │                │
┌───────▼─────────────▼────────────────▼─────────────┐
│              意图识别层（LLM）                       │
│                                                     │
│  输入：原始帖子/评论                                │
│  输出：{                                            │
│    intent: "需要辅助工具",                          │
│    game: "王者荣耀",                                │
│    pain: "卡顿严重",                               │
│    score: 0.87  // 转化概率                         │
│  }                                                  │
└────────────────────┬────────────────────────────────┘
                     │ score > 0.7 进入CRM
┌────────────────────▼────────────────────────────────┐
│              线索管理层（CRM）                       │
│  去重 │ 优先级排序 │ 分配状态 │ 跟进记录           │
└──────┬────────────────────────┬────────────────────-┘
       │                        │
┌──────▼──────┐          ┌──────▼──────┐
│  自动触达   │          │  人工跟进   │
│  score>0.85 │          │  score<0.85 │
│             │          │             │
│ 生成个性化  │          │ 推送运营    │
│ 回复话术    │          │ 人员处理    │
│ 自动发送    │          └─────────────┘
└──────┬──────┘
       │
┌──────▼──────────────────────────────────────────────┐
│              转化追踪层                              │
│  点击 → 注册 → 付费 → 归因分析                     │
│  自动计算每条渠道 ROI                               │
└─────────────────────────────────────────────────────┘
```

### 采集配置
```yaml
keywords:
  - "辅助工具推荐"
  - "脚本哪里买"
  - "账号出售"
  - "想买个号"
  - "有没有好用的助手"
  - "有没有好用的脚本"
  - "求推荐游戏助手"

games:
  - 王者荣耀
  - DNF手游
  - 格斗山海
  - 三角洲行动
  - 泰拉瑞亚

platforms:
  - NGA论坛
  - 百度贴吧
  - 5173
  - 交易猫评论区
  - 游民星空
```

### 自动触达话术规则
```
输入：{用户痛点, 游戏名, 818对应产品}
输出：个性化回复，不超过100字，自然口吻，含产品链接

规则：
- 不暴露是机器人
- 每个平台账号每日触达上限：50条
- 相同用户48小时内不重复触达
- 触达后3天无响应自动归档
```

### 数据库设计
```sql
t_lead             -- 线索表（来源、意图、评分、状态）
t_lead_source      -- 来源渠道配置
t_outreach         -- 触达记录
t_conversion       -- 转化追踪
t_keyword_config   -- 关键词配置
t_platform_account -- 触达用账号池
```

## 五、风险与对策

| 风险 | 对策 |
|------|------|
| AI中转站：上游API涨价 | 多供应商路由，自动切换成本最优 |
| AI中转站：用户滥用 | Token限流 + 违规检测 + 封号机制 |
| 获客Agent：平台封号 | 账号池轮换 + 行为模拟 + 频率控制 |
| 获客Agent：合规风险 | 内容审核 + 只触达公开帖子 + 不发私信 |
| 两者：资金安全 | 充值走818现有支付体系，资金隔离 |
