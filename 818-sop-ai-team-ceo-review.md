## 二、CEO 补充 SOP

### CEO-01 资金安全与 USDT 提现流程 🔴

```
原则：
  - 任何金额的提现，Albert 必须二次确认
  - 热钱包余额上限：$500（超过自动转冷钱包）
  - 月度对账：收入对比充值记录，误差<0.1%

USDT 收入管理：
  ┌─────────────────────────────────────┐
  │  用户充值 → 收款热钱包（每单独立地址）│
  │       ↓ 每日自动归集               │
  │  运营热钱包（日常运营资金）         │
  │       ↓ 余额>$500 自动转移          │
  │  冷钱包（Albert保管私钥，离线）     │
  └─────────────────────────────────────┘

提现申请流程（从热钱包取出）：
  1. 提交：说明金额 + 用途 + 收款地址
  2. Albert 核对钱包余额与账目
  3. Albert 签名发送（冷钱包需硬件设备）
  4. 记录：{日期, 金额, 用途, TxHash}
  5. 禁止：任何人单独掌控热钱包私钥

月度财务对账（每月1日）：
  □ USDT 链上收入 vs 系统充值记录
  □ 用户余额总量 vs 热钱包余额（负债对等）
  □ 月净利润 = 收入 - 上游API成本
  □ 记录存档（后期税务/审计使用）
```

### CEO-02 法律合规检查点 🔴

```
俄罗斯市场进入前必查（One-time）：
□ 确认：中转AI API服务是否违反俄罗斯联邦法律
□ 确认：USDT收款在俄罗斯是否合规
□ 确认：Telegram Bot商业使用条款限制
□ 确认：公司主体是否需要在俄罗斯注册
→ 建议：花费$200找专业律师出具意见书

中国主体合规（持续）：
□ ICP证范围是否覆盖AI API中转业务
□ 境外USDT收入是否需要申报
□ 个人信息保护法：俄罗斯用户数据存储位置合规

触发重新评估的条件：
  - 俄罗斯出台新AI监管法规
  - 中国出台境外AI服务新规
  - Telegram修改商业Bot政策
  → 触发时：暂停新用户注册，7天内完成评估
```

### CEO-03 关键人员风险应急 🔴

```
场景：资深研发离职或突然无法工作

预防措施（平时执行）：
□ 所有核心模块有文档（README + 设计说明）
□ 支付/计费核心代码，Albert能读懂（不一定会写）
□ 运维账号（服务器/数据库/云服务）Albert独立掌握
□ 每季度：Albert跑一次完整部署流程，不依赖研发

触发应急（研发无法工作）：
  第1天：
    - Albert接管服务器监控
    - 暂停非关键功能发布
    - 评估当前系统稳定性

  第1周：
    - 招聘临时外包（预算¥500/天）
    - 确定核心代码交接文档
    - 通知用户可能的服务调整

  长期：
    - 招聘新研发（2周内）
    - 考虑：Go/Python关键模块是否引入Co-pilot辅助

账号安全（离职时必做）：
□ 立即回收：GitHub权限 / 服务器SSH Key / 数据库账号
□ 修改：所有共享密码（API Key / 云账号）
□ 时效：离职当天完成，不超过24小时
```

### CEO-04 上游供应商中断应急 🟠

```
场景A：OpenAI 封号 / 提高价格 >30%

立即响应（4小时内）：
  1. 评估：占总调用量比例
  2. 调整路由权重：DeepSeek/Claude 承接流量
  3. 更新 model_config.yaml（热更新，无需停服）
  4. 评估成本影响：是否需要调整用户定价

若OpenAI永久封号：
  - 48小时内：全量迁移到Claude + DeepSeek
  - 通知用户：GPT-4o暂时不可用，替代方案说明
  - 补偿：受影响用户发放10%余额补偿

场景B：Claude/所有供应商同时中断

触发服务降级模式：
  - Bot回复：「服务临时维护，预计xxx恢复」
  - 停止新用户充值（防止资损）
  - 优先恢复最赚钱的模型

供应商多样化原则：
  - 单一供应商占比不超过 60%
  - 至少保持 3 个可用供应商
  - 每季度测试备用供应商切换演练
```

### CEO-05 用户危机公关 SOP 🟠

```
触发条件：
  - 服务宕机 > 30 分钟
  - 用户资金损失（多扣/漏扣）
  - 数据安全事件
  - 负面内容在Telegram社群扩散

响应原则：
  快 > 完美（30分钟内发声，哪怕只说"我们知道了")
  承认 > 掩盖（用户能接受故障，不接受欺骗）
  补偿 > 道歉（行动比话语有力）

话术模板（Telegram Bot 群发）：
  [服务通知]
  我们注意到部分用户遇到 [问题描述]。
  原因：[如已知，否则写"正在排查"]
  预计恢复：[时间，或"我们会尽快更新"]
  补偿方案：[受影响用户 +X% 余额]
  感谢理解，818 团队

发布审批：
  - Albert 亲自审批所有对外公告
  - 不允许运营自行发布危机声明
  - 发布后30分钟跟进一次进展

事后：
  □ 24小时内发布事后说明（根因 + 措施）
  □ 补偿自动发放（系统执行，不需要用户申请）
  □ 复盘记录存档
```

### CEO-06 战略决策门控 🟡

```
什么需要 Albert 拍板（不授权）：

产品决策：
  - 新产品立项
  - 现有产品下线
  - 定价调整 >10%
  - 向俄罗斯以外市场扩张

财务决策：
  - 单笔支出 > ¥5000
  - 新的供应商合同
  - USDT 提现 > $200

人事决策：
  - 招聘
  - 离职处理

月度战略Review（每月最后一个周五，1小时）：
  □ 核心指标 vs OKR 进度
  □ 本月收入 vs 目标（差距原因）
  □ 下月优先级排序（只确认Top3）
  □ 是否需要调整产品方向

产品 Go/No-Go 标准（上线30天后评估）：
  继续投入：付费用户 > 50 且月营收 > $2000
  暂停迭代：付费用户 < 20 或月营收 < $500
  关闭产品：连续2个月达到暂停标准
```

### CEO-07 竞争情报监控 🟡

```
月度竞品扫描（每月1次，运营执行）：

监控对象：
  - 俄罗斯主要AI中转站（已知3-5家）
  - OpenAI/Anthropic官方俄罗斯政策变化
  - Telegram Bot AI服务新进入者

检查内容：
  □ 竞品定价是否有变化
  □ 竞品是否有新功能
  □ 社群口碑（Telegram群讨论）
  □ 是否有大资本进入

输出：
  简报发给 Albert（不超过1页）
  格式：{变化点, 对我们的影响, 建议行动}

触发立即响应的信号：
  - 竞品降价 > 20%
  - 大厂（腾讯/字节）进入俄罗斯AI代理市场
  - OpenAI宣布正式支持俄罗斯市场
  → 触发：Albert 48小时内做战略调整决策
```
